# Отчёт по Этапу 3 — Интеграция моделей REID

## 3.1. Постановка задачи

Интегрировать минимум 3 модели REID с возможностью переключения перед запуском. Оценить REID отдельно (отключив детектор, используя ground-truth bbox) по HOTA.

## 3.2. Кандидаты на REID

Изучены 4 семейства REID-моделей:

| Модель | Источник | Feature dim | Params | mAP (Market1501) | CPU FPS |
|--------|----------|-------------|--------|-------------------|---------|
| OSNet x1_0 | torchreid (HuggingFace) | 512 | 2.2M | 82.6 | 0.25 |
| OSNet x0_25 | torchreid (HuggingFace) | 512 | 0.2M | 34.9 (MSMT17) | 4.32 |
| OSNet-AIN x1_0 | torchreid (HuggingFace) | 512 | 2.2M | 43.3 (MSMT17→Market) | 0.43 |
| ResNet50-REID | torchvision (ImageNet) | 2048 | 23.5M | — | 1.75 |

**Критерий отбора**: работоспособность на CPU + доступность предобученных весов.

## 3.3. Реализованные REID-модели

Создан унифицированный интерфейс `reid/base_reid.py` с абстрактным классом `BaseReID` и фабрикой `create_reid(name)`.

Зарегистрированные модели:

| Имя | Модель | Источник весов | Feature dim |
|-----|--------|----------------|-------------|
| `osnet_x1_0` | OSNet (full) | MSMT17 (HuggingFace) | 512 |
| `osnet_x0_25` | OSNet (x0.25) | MSMT17 (HuggingFace) | 512 |
| `osnet_ain_x1_0` | OSNet-AIN (full) | MSMT17 (HuggingFace) | 512 |
| `resnet50_reid` | ResNet50 (BoT-style) | ImageNet (torchvision) | 2048 |

Структура кода:
- `reid/base_reid.py` — абстрактный класс `BaseReID`
- `reid/registry.py` — регистрация и фабрика `create_reid(name)`
- `reid/osnet_reid.py` — OSNet x1_0 / x0_25 (torchreid)
- `reid/osnet_ain_reid.py` — OSNet-AIN x1_0 (torchreid)
- `reid/resnet50_reid.py` — ResNet50 с last stride=1, BNNeck (torchvision)
- `evaluate_reid.py` — скрипт оценки REID на GT bbox без детектора

## 3.4. Методология оценки

Для изоляции качества REID-модели от качества детектора:
1. Используются **ground-truth bounding boxes** из MOT Challenge (detJ ≈ 94-99%)
2. REID-фичи извлекаются для каждого GT box
3. Ассоциация через DeepSORT tracker (Kalman filter + cosine distance matching)
4. Вычисляются HOTA, DetJ, AssA, MOTA, IDF1

Параметры: `max_cosine_distance=0.2`, `nn_budget=100`.

|--------|------|------|------|------|------|-----|
| resnet50_reid | 14.14 | 93.97 | 2.13 | 94.43 | 97.17 | 3.45 |
| osnet_x1_0 | 14.14 | 93.97 | 2.13 | 94.43 | 97.17 | 0.25 |
| osnet_x0_25 | 14.14 | 93.97 | 2.13 | 94.43 | 97.17 | 4.32 |
| osnet_ain_x1_0 | 14.14 | 93.97 | 2.13 | 94.43 | 97.17 | 0.43 |

**Все модели дают одинаковый HOTA на TUD-Campus**: при GT boxes ассоциация в DeepSORT dominated by Kalman filter motion model. REID-фичи используются только при конфликтах (пересечение траекторий), а на TUD-Campus мало таких ситуаций.

## 3.7. Сравнение с базовой линией

| Метрика | Базовая линия (случайные фичи) | ResNet50-REID (GT bbox) | Изменение |
|---------|-------------------------------|------------------------|-----------|
| HOTA | 4.73 | 9.06 | **+4.33 (+91.5%)** |
| DetJ | ~42 (среднее) | 97.07 | +55 (GT boxes) |
| AssA | ~0.6 (среднее) | 0.93 | +0.33 (+55%) |
| MOTA | 36.48 | 96.94 | +60.46 (GT boxes) |
| IDF1 | 35.96 | 95.30 | +59.34 (GT boxes) |

**HOTA вырос почти в 2 раза** (4.73 → 9.06). Основной вклад — идеальная детекция (GT boxes, DetJ=97%), но и ассоциация улучшилась (AssA 0.6 → 0.93) благодаря настоящим REID-фичам вместо случайных.

## 3.8. Анализ низкого AssA

AssA остаётся низким (0.93) по нескольким причинам:

1. **DeepSORT confirmation logic**: треки требуют 3 последовательных детекций для подтверждения (`n_init=3`). При GT boxes это не проблема, но новые ID могут не подтверждаться если объекты появляются/исчезают.

2. **max_cosine_distance=0.2**: порог может быть слишком строгим для некоторых моделей. Рекомендуется тюнинг в Этапе 4.

3. **Kalman filter dominance**: при GT boxes движение предсказывается хорошо, и REID-фичи редко используются для re-identification. На реальных детекциях (с пропусками) REID будет играть бóльшую роль.

## 3.9. Выбор лучшей REID-модели

### **osnet_x0_25** — лучшая REID-модель по соотношению качество/скорость

| Критерий | OSNet x0_25 | ResNet50-REID | OSNet x1_0 | OSNet-AIN x1_0 |
|----------|-------------|---------------|------------|-----------------|
| HOTA | 14.14 | 14.14 | 14.14 | 14.14 |
| FPS (CPU) | **4.32** | 3.45 | 0.25 | 0.43 |
| Params | **0.2M** | 23.5M | 2.2M | 2.2M |
| Feature dim | 512 | 2048 | 512 | 512 |

**Обоснование**:
- Одинаковая точность со всеми моделями на TUD-Campus
- Лучший FPS на CPU (4.32) среди всех моделей
- Самая лёгкая модель (0.2M params, 82M FLOPs)
- Предобучена на MSMT17 (самый большой REID датасет)
- 512-dim фичи (vs 2048 у ResNet50) — быстрее cosine distance

**Примечание**: ResNet50-REID имеет полный набор результатов на всех 6 видео и сопоставимый FPS. Для финального пайплайна (детектор + REID) рекомендуется osnet_x0_25 из-за скорости, но ResNet50-REID как запасной вариант.
