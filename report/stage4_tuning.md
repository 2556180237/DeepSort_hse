# Отчёт по Этапу 4 — Настройка параметров и интеграция

## 4.1. Постановка задачи

Объединить лучший детектор (YOLO11s) и лучшую REID-модель (osnet_x0_25) в полном пайплайне DeepSORT. Подобрать параметры для максимизации HOTA при сохранении FPS ≥ 5 (на Colab с GPU). HOTA должен быть выше базовой линии на каждом видео.

## 4.2. Полный пайплайн

Создан скрипт `evaluate_full_pipeline.py` для оценки HOTA/MOTA/IDF1/FPS полного пайплайна:

```
Изображение → YOLO11s (детекция) → OSNet x0_25 (REID-фичи) → DeepSORT (трекинг) → Результат
```

Параметры DeepSORT, доступные для тюнинга:

| Параметр | Описание | Default |
|----------|----------|---------|
| `conf_threshold` | Порог уверенности детектора | 0.3 |
| `nms_max_overlap` | Порог NMS | 1.0 |
| `max_cosine_distance` | Порог cosine distance для ассоциации | 0.2 |
| `nn_budget` | Размер галереи REID-фичей | 100 |
| `max_age` | Макс. пропусков до удаления трека | 30 |
| `n_init` | Детекций для подтверждения трека | 3 |
| `max_iou_distance` | Порог IoU для matching cascade | 0.7 |

### 4.3.2. Анализ экспериментов

**tune1 (conf=0.2, cos=0.3, n_init=2, max_age=50)** — худший результат:
- Снижение `conf_threshold` до 0.2 привело к большому количеству FP (ложных детекций)
- DetJ упал до 52.47 (vs 57.36 default), MOTA до 47.67
- Вывод: слишком низкий порог детекции вреден

**tune2 (conf=0.4, cos=0.15, budget=50)** — лучший HOTA:
- Повышение `conf_threshold` до 0.4 уменьшило FP, повысило точность
- Снижение `max_cosine_distance` до 0.15 сделало ассоциацию более строгой
- HOTA=6.63 (лучший), но IDF1=65.93 (ниже default)

**tune3 (conf=0.35, n_init=1, max_age=20)** — агрессивный трекинг:
- `n_init=1` подтверждает треки сразу (быстрее, но больше FP в треках)
- `max_age=20` быстрее удаляет потерянные треки
- HOTA=6.56, но FPS=2.11 (худший)

**tune4 (conf=0.4, cos=0.2, default rest)** — лучший баланс:
- Повышение `conf_threshold` до 0.4 — меньше FP
- Остальные параметры по умолчанию
- HOTA=6.61, MOTA=61.18 (лучший), IDF1=67.94 (лучший)

## 4.6. Индивидуальная настройка по видео

Анализ показывает, что основные проблемы сосредоточены на двух видео:

**MOT16-09 (HOTA=3.98, DetJ=39.70)**: низкий DetJ из-за отдалённых/маленьких людей. Детектор пропускает много объектов. Возможные улучшения: увеличение `img_size` (640→1280), но это замедлит инференс в 2-3 раза.

**MOT16-11 (HOTA=4.80, DetJ=46.68)**: аналогичная проблема — большое количество людей, многие далеко от камеры.

Для остальных видео (TUD-Campus, TUD-Stadtmitte, KITTI-17, PETS09-S2L1) параметры работают хорошо. Индивидуальная настройка не требуется — проблема в детекторе, а не в параметрах трекера.
