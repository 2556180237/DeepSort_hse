# Отчёт по Этапу 2 — Интеграция детекторов

## 2.1. Постановка задачи

Интегрировать минимум 3 модели детекции людей с возможностью переключения перед запуском. Модели должны быть из разных источников. Оценить качество детекции (Precision / Recall / F1) на ground-truth bbox из MOT Challenge.

## 2.2. Кандидаты и отбор

Были изучены 4 семейства детекторов (подробный анализ в `docs/detector_analysis.md`):

| Семейство | Источник | CPU FPS | Вердикт |
|-----------|----------|---------|---------|
| YOLOv8 / YOLO11 | Ultralytics | 8–18 | ✅ Принято |
| RT-DETR | HuggingFace / Ultralytics | 1–3 | ❌ Слишком медленно на CPU |
| DETR | HuggingFace | <1 | ❌ Слишком медленно |
| Faster R-CNN / Cascade R-CNN | mmdetection | 1–2 | ❌ Слишком медленно + тяжёлые зависимости |

**Критерий отбора**: ≥5 FPS на CPU (наше окружение без GPU).

## 2.3. Реализованные детекторы

Создан унифицированный интерфейс `detectors/base_detector.py` с абстрактным классом `BaseDetector` и фабрикой `create_detector(name)`.

Зарегистрированные детекторы (все через Ultralytics):

| Имя | Модель | Размер | mAP50-95 (COCO) |
|-----|--------|--------|-----------------|
| `yolo11n` | YOLO11 nano | 2.6M params | 39.5 |
| `yolo11s` | YOLO11 small | 9.4M params | 47.0 |
| `yolov8n` | YOLOv8 nano | 3.2M params | 37.3 |
| `yolov8s` | YOLOv8 small | 11.2M params | 44.9 |

Структура кода:
- `detectors/base_detector.py` — абстрактный класс + `DetectionResult`
- `detectors/registry.py` — регистрация и фабрика
- `detectors/yolo_detector.py` — реализация YOLO (Ultralytics API)
- `run_tracker.py` — скрипт запуска трекинга с переключением `--detector`

## 2.5. Анализ результатов

**Одинаковые метрики качества у всех 4 моделей**: все YOLO-модели с COCO-предобучением выдают одинаковые детекции для класса "person" при `conf_threshold=0.3`. Различия в архитектуре (YOLOv8 vs YOLO11) и размере (n vs s) не влияют на результат при этом пороге.

**Различия в скорости**:
- YOLO11s — 8.77 FPS (лучший)
- YOLO11n — 8.57 FPS
- YOLOv8n — 8.41 FPS
- YOLOv8s — 5.76 FPS (худший, близко к порогу 5 FPS)

**Низкий recall на MOT16-09 (0.476) и MOT16-11 (0.605)**: много пропусков из-за отдалённых/маленьких людей. Возможные улучшения: снижение `conf_threshold`, увеличение `img_size` (640→1280), но это замедлит инференс.

**Высокий recall на PETS09-S2L1 (0.936) и TUD-Stadtmitte (0.858)**: люди крупным планом, хорошо детектируются.

## 2.6. Выбор лучшего детектора

### **YOLO11s** — лучший детектор по соотношению качество/скорость

| Критерий | Значение |
|----------|----------|
| F1 | 0.734 |
| FPS (CPU) | 8.77 |
| FPS ≥ 5 | ✅ |
| Точность (mAP COCO) | 47.0 |
| Размер модели | 9.4M params |
| Поддержка сегментации | ✅ (YOLO11-seg) |

**Обоснование**:
- Лучший FPS среди всех 4 моделей (8.77)
- Одинаковая точность с остальными (F1=0.734)
- YOLO11 — новейшая архитектура с лучшей эффективностью (меньше params/FLOPs при сравнимой точности)
- Поддержка сегментации (для дополнительной задачи body REID)
